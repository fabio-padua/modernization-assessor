# v0.1 — 2-week MVP scope

Goal: a working end-to-end thin slice that proves the architecture and produces a real (if rough) modernization assessment from a sample inventory.

## In scope

- **1 Foundry project** (created via `azd up`) with **GPT-4.1** deployment
- **MAF orchestrator** in C# .NET 9, running locally (ACA wiring in v0.2)
- **2 of 5 specialist agents** end-to-end:
  - **Discovery** — accepts an Azure Migrate-shaped CSV, normalizes
  - **Workload Classifier** — emits IaaS/PaaS/SaaS/retire recommendation per app with rationale
- **1 MCP tool**: `AzurePricingTool` (calls Azure Retail Prices API)
- **Sample customer inventory** (synthetic) in `samples/inventory/`
- **Eval harness** wired into `.foundry/` per skill standard
- **Output**: a JSON assessment + a Markdown summary

## Out of scope (v0.2+)

- ACA deployment, APIM gateway, Cosmos DB persistence
- AI Search RAG (WAF + migration KB)
- Cost Estimator, Migration Planner, Risk Assessment agents
- Partner branding / multi-tenancy
- Excel/PPT output (will use Markdown for v0.1)

## Acceptance criteria

1. `dotnet run --project src/Orchestrator -- --inventory samples/inventory/contoso.csv` produces:
   - `out/assessment-<timestamp>.json` with normalized inventory + per-app classification
   - `out/assessment-<timestamp>.md` with executive summary
2. Evaluation suite runs against 5 golden inventories with classifier accuracy ≥ 80% on labeled set.
3. README quickstart works on a clean machine (.NET 9 + azd + Docker).
4. Trace data visible in Application Insights with `agentName`, `toolName` tags.
